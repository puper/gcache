# 需求与优化范围

## 背景
当前 `v2` 目录存在核心实现缺失（`newShardedCache`、`shardedCache`），导致无法构建与验证性能；TTL 与哈希路径也存在可优化空间。

## 本次目标
1. 恢复缓存核心实现，保证基础功能可用。
2. 降低 TTL 清理开销与并发锁竞争。
3. 优化哈希通用路径，减少 unsafe 与分配开销。
4. 支持分片数可配置，提升不同机器上的可调优性。
5. 提升 benchmark 场景代表性，覆盖更接近真实业务的 key 分布和读写混合负载。

## 影响面
- 核心缓存实现：新增分片缓存结构与方法。
- 构建器：新增分片配置项与自动分片策略。
- 过期结构：时间轮/队列实现替换。
- 哈希模块：泛型 key 哈希逻辑调整。
- 基准测试：新增与调整测试负载模型。
